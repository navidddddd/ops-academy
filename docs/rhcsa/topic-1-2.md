# 🐧 درس ۱.۱: دسترسی به Shell و اجرای دستورات

:::warning 💡 یادداشت معمار ارشد
سلام داوطلب گرامی. من به عنوان ممتحن رسمی RHCSA و ارشد معماری لینوکس، آمادگی شما را برای این دوره فشرده ارزیابی می‌کنم. سوابق شما در مدیریت شبکه ارزشمند است، اما در ترمینال لینوکس، ما از صفر مطلق شروع خواهیم کرد. در این آزمون هیچ بهانه‌ای پذیرفته نیست؛ جزئیات و استانداردها در محیط‌های Production حیاتی هستند.
:::

## 🧠 ۱. مفهوم اصلی (The Core Concept)

در محیط‌های سازمانی و سرورهای لینوکس (مانند Red Hat Enterprise Linux)، هیچ واسط گرافیکی (GUI) وجود ندارد. تمام کارهای یک Linux Architect یا SRE از طریق `shell prompt` انجام می‌شود.

واژه **Shell** در واقع یک مفسر فرمان (Command Interpreter) است که دستورات متنی شما را دریافت کرده، آن‌ها را ترجمه می‌کند و به Kernel سیستم‌عامل تحویل می‌دهد. فرمت استاندارد و ساختار صحیح دستورات در لینوکس همیشه از یک الگوی مشخص پیروی می‌کند:

```text
Command [-options] [arguments]

```

* **`Command`**: نام برنامه یا ابزاری است که می‌خواهید اجرا کنید.
* **`Options`**: رفتار دستور را تغییر می‌دهند (معمولاً با یک خط تیره `-` برای حالت تک‌حرفی یا دو خط تیره `--` برای کلمات کامل شروع می‌شوند).
* **`Arguments`**: هدفی است که دستور روی آن اعمال می‌شود (مانند مسیر یک فایل).

:::danger ⚠️ مهم‌ترین اصل لینوکس
لینوکس به شدت به کوچک و بزرگ بودن حروف (Case-sensitive) حساس است. دستور `ls` با `LS` یا `Ls` کاملاً متفاوت است و سیستم آن‌ها را نمی‌شناسد.
:::

## 💻 ۲. آزمایشگاه عملی (Hands-on Lab)

وارد ماشین مجازی خود شوید، یک Terminal باز کنید و دستورات زیر را به ترتیب اجرا و تحلیل کنید:

### پاک‌سازی محیط کار

```bash
clear

```

دستور `clear` ابزار اصلی است که تمام متن‌های موجود در صفحه ترمینال شما را پاک می‌کند تا یک محیط کاری تمیز داشته باشید. این دستور نیازی به آرگومان ندارد.

### چاپ مسیر فعلی

```bash
pwd

```

مخفف Print Working Directory است. وظیفه آن چاپ کردن مسیر دقیقی است که در حال حاضر در آن قرار دارید.

### لیست کردن جزئیات دایرکتوری

```bash
ls -l /var

```

* **`ls`**: دستور اصلی برای لیست کردن محتویات.
* **`-l`**: یک آپشن (Long listing format) برای نمایش فایل‌ها با جزئیات کامل (دسترسی‌ها، مالک، حجم).
* **`/var`**: یک آرگومان که به دستور اعلام می‌کند محتویات پوشه سیستم `/var` را نمایش دهد.

### جابه‌جایی در مسیرها

```bash
cd /etc

```

مخفف Change Directory برای جابه‌جایی بین پوشه‌ها در لینوکس است. با این کار محیط ترمینال شما به پوشه `/etc` منتقل می‌شود.

### چاپ در خروجی

```bash
echo "Welcome to RHCSA"

```

دستور `echo` برای چاپ متنی است که به آن داده می‌شود. عبارت `"Welcome to RHCSA"` یک آرگومان متنی (String) است.

## 🧠 ۱. مفهوم اصلی (The Core Concept)

در لینوکس یک اصل معروف وجود دارد: **Everything is a file** (همه چیز یک فایل است). وقتی شما یک دستور را اجرا می‌کنید، سیستم‌عامل ۳ کانال یا جریان داده (Data Stream) برای آن باز می‌کند که به آن‌ها File Descriptor (FD) می‌گویند:

* **`0` (Standard Input - stdin)**: ورودی دستور که معمولاً از کیبورد گرفته می‌شود.
* **`1` (Standard Output - stdout)**: خروجی استاندارد دستور که روی صفحه نمایش داده می‌شود.
* **`2` (Standard Error - stderr)**: پیام‌های خطا که به دلیل عدم اجرای درست تولید می‌شوند.

شما به عنوان مهندس SRE می‌توانید مسیر این جریان‌ها را تغییر دهید (Redirection) تا آن‌ها در فایل ذخیره شوند یا از طریق یک لوله (Pipe) به دستور دیگری شلیک شوند.

## 💻 ۲. آزمایشگاه عملی (Hands-on Lab)

دستورات زیر را خط به خط در ترمینال خود وارد کرده و رفتار آن‌ها را تحلیل کنید:

### اپراتور بازنویسی (Overwrite)
```bash
echo "Red Hat Enterprise Linux" > os.txt

```

علامت `>` خروجی استاندارد را می‌گیرد، اگر فایل `os.txt` وجود نداشته باشد آن را می‌سازد و اگر وجود داشته باشد، محتوای قبلی را کاملاً پاک کرده و متن جدید را جایگزین می‌کند.

### اپراتور الحاق (Append)

```bash
echo "Version 9" >> os.txt

```

علامت `>>` محتوای قبلی فایل را تخریب نمی‌کند، بلکه متن جدید را به انتهای آن اضافه می‌کند.

### تایید با خواندن فایل

```bash
cat os.txt

```

دستور `cat` (مخفف Concatenate) ابزاری برای خواندن و نمایش محتویات فایل‌ها در ترمینال است.

### هدایت اختصاصی خطاها

```bash
ls /root/secret 2> error.log

```

علامت `2>` خطاهای تولید شده (stderr) را شکار می‌کند و می‌گوید: "اگر دستوری خطا داد، آن را چاپ نکن، بلکه داخل فایل `error.log` بریز".

### ترکیب دستورات با پایپ

```bash
ls -l /etc | wc -l

```

علامت `|` (Pipe) خروجی دستور اول (`ls`) را مستقیماً به عنوان ورودی به دستور سمت راست یعنی `wc -l` (شمارنده خطوط) متصل می‌کند.

## ⚠️ ۳. چالش هرج‌و‌مرج (The Chaos Challenge)

یک اسکریپت مانیتورینگ قدیمی در سرور دچار مشکل شده و لاگ‌های خطا با لاگ‌های معمولی مخلوط می‌شوند. شما باید این وضعیت را ساماندهی کنید.

**وظایف شما:**
۱. دستوری بنویسید که مسیر جاری شما (`pwd`) را گرفته و در فایلی به نام `report.txt` ذخیره کند (محتوای قبلی باید پاک شود).
۲. بدون پاک شدن محتوای قبلی، متن `RHCSA Training` را به انتهای همان فایل `report.txt` اضافه کنید.
۳. دستوری بنویسید که تلاش کند محتویات یک پوشه فرضی به نام `/not-exist` را لیست کند، اما خروجیِ خطای آن مستقیماً در فایلی به نام `missing.log` ذخیره گردد.
۴. با استفاده از Pipe، دستوری بنویسید که محتویات پوشه `/etc` را لیست کرده و با کمک دستور `grep` فقط خطوطی را خروجی بدهد که کلمه `network` در آن‌ها وجود دارد.
