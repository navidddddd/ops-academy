# 🐧 درس ۱.۱: دسترسی به `Shell` و اجرای دستورات

:::warning 💡 یادداشت معمار ارشد
سلام داوطلب گرامی. من به عنوان ممتحن رسمی RHCSA و ارشد معماری لینوکس، آمادگی شما را برای این دوره فشرده ارزیابی می‌کنم. در ترمینال لینوکس، ما از صفر مطلق شروع خواهیم کرد. در این مسیر هیچ بهانه‌ای پذیرفته نیست؛ جزئیات و استانداردها در محیط‌های Production حیاتی هستند.
:::

## 🧠 مفهوم اصلی (The Core Concept)

در محیط‌های سازمانی و سرورهای لینوکس (مانند Red Hat Enterprise Linux)، هیچ واسط گرافیکی (GUI) وجود ندارد. تمام کارهای یک Linux Architect از طریق `shell prompt` انجام می‌شود.

واژه **Shell** در واقع یک مفسر فرمان (Command Interpreter) است که دستورات متنی شما را دریافت کرده، آن‌ها را ترجمه می‌کند و به Kernel سیستم‌عامل تحویل می‌دهد. ساختار صحیح دستورات (Syntax) در لینوکس همیشه از یک الگوی مشخص پیروی می‌کند:

```bash
Command [-options] [arguments]

```

* **`Command`**: نام برنامه یا ابزاری است که می‌خواهید اجرا کنید.
* **`Options`**: رفتار دستور را تغییر می‌دهند (معمولاً با یک خط تیره `-` برای حالت تک‌حرفی یا دو خط تیره `--` برای کلمات کامل شروع می‌شوند).
* **`Arguments`**: هدفی است که دستور روی آن اعمال می‌شود (مانند مسیر یک دایرکتوری).

:::danger ⚠️ مهم‌ترین اصل لینوکس
لینوکس به شدت به کوچک و بزرگ بودن حروف (Case-sensitive) حساس است. دستور `ls` با `LS` کاملاً متفاوت است و سیستم دومی را نمی‌شناسد.
:::

## 💻 آزمایشگاه عملی (Hands-on Lab)

وارد ماشین مجازی خود شوید، یک Terminal باز کنید و دستورات زیر را به ترتیب اجرا و تحلیل کنید:

### ۱. پاک‌سازی محیط کار

```bash
clear

```

این ابزار تمام متن‌های موجود در صفحه ترمینال شما را پاک می‌کند تا یک محیط کاری تمیز داشته باشید. این دستور در حالت عادی نیازی به آپشن یا آرگومان ندارد.

### ۲. چاپ مسیر فعلی

```bash
pwd

```

مخفف Print Working Directory است. وظیفه آن چاپ کردن مسیر دقیقی است که در حال حاضر در آن قرار دارید.

### ۳. لیست کردن جزئیات دایرکتوری

```bash
ls -l /var

```

* بخش `ls`: دستور اصلی برای لیست کردن محتویات.
* بخش `-l`: یک آپشن برای نمایش فایل‌ها با جزئیات کامل (دسترسی‌ها، مالک، حجم).
* بخش `/var`: یک آرگومان که به دستور اعلام می‌کند محتویات این پوشه خاص را نمایش دهد.

### ۴. جابه‌جایی در مسیرها

```bash
cd /etc

```

دستور `cd` مخفف Change Directory است. با این کار محیط ترمینال شما به پوشه `/etc` منتقل می‌شود.

### ۵. چاپ در خروجی

```bash
echo "Welcome to RHCSA"

```

دستور `echo` برای چاپ متنی است که به آن داده می‌شود. عبارت داخل کوتیشن، آرگومانِ متنیِ این دستور است.

## 🎯 چالش هرج‌و‌مرج (The Chaos Challenge)

یک سناریوی واقعی در یک سرور بحران‌زده: شما به عنوان مدیر سیستم وارد سرور شده‌اید و باید اطلاعات سریعی از وضعیت دایرکتوری‌ها به دست آورید.

**وظایف شما:**

۲. بدون اینکه مسیر فعلی خود را تغییر دهید (بدون `cd`)، دستوری بنویسید که محتویات دایرکتوری `/boot` را با تمام جزئیات ساختاری نشان دهد.
۳. دستور `LS -L /var` را در ترمینال اجرا کنید. دقیقاً چه خطایی دریافت کردید؟ دلیل بروز این خطا را تحلیل کنید.

## 🧠 مفهوم اصلی (The Core Concept)

در لینوکس یک اصل معروف وجود دارد: **Everything is a file** (همه چیز یک فایل است). وقتی شما یک دستور را اجرا می‌کنید، سیستم‌عامل ۳ کانال داده (Data Stream) برای آن باز می‌کند که به آن‌ها `File Descriptor` می‌گویند:

* **شناسه `0` (stdin)**: ورودی دستور که معمولاً از کیبورد گرفته می‌شود.
* **شناسه `1` (stdout)**: خروجی موفقیت‌آمیز دستور که روی صفحه نمایش داده می‌شود.
* **شناسه `2` (stderr)**: پیام‌های خطایی که به دلیل عدم اجرای درست دستور تولید می‌شوند.

شما به عنوان مهندس سیستم می‌توانید مسیر این جریان‌ها را تغییر دهید (Redirection) تا آن‌ها را در یک فایل لاگ ذخیره کنید، یا خروجیِ یک دستور را از طریق لوله (`Pipe`) به دستور دیگری شلیک کنید.

## 💻 آزمایشگاه عملی (Hands-on Lab)

دستورات زیر را خط به خط در ترمینال خود وارد کرده و رفتار آن‌ها را تحلیل کنید:

### ۱. اپراتور بازنویسی (Overwrite)
```bash
echo "Red Hat Enterprise Linux" > os.txt

```

علامت `>` خروجی استاندارد را می‌گیرد. اگر فایل وجود نداشته باشد آن را می‌سازد و اگر وجود داشته باشد، محتوای قبلی را کاملاً پاک کرده و متن جدید را جایگزین می‌کند.

### ۲. اپراتور الحاق (Append)

```bash
echo "Version 9" >> os.txt

```

علامت `>>` محتوای قبلی فایل را تخریب نمی‌کند، بلکه متن جدید را به انتهای آن اضافه می‌کند.

### ۳. تایید با خواندن فایل

```bash
cat os.txt

```

دستور `cat` (مخفف Concatenate) ابزاری برای خواندن و نمایش سریع محتویات فایل‌های متنی در ترمینال است.

### ۴. هدایت اختصاصی خطاها

```bash
ls /root/secret 2> error.log

```

علامت `2>` خطاهای تولید شده (`stderr`) را شکار می‌کند. این اپراتور می‌گوید: "اگر دستوری خطا داد، خطا را روی صفحه چاپ نکن، بلکه آن را داخل فایل متنی ذخیره کن."

### ۵. ترکیب دستورات با پایپ

```bash
ls -l /etc | wc -l

```

علامت `|` (Pipe) ابزار بسیار قدرتمندی است که خروجی دستور اول (`ls`) را مستقیماً به عنوان ورودی به دستور سمت راست یعنی `wc -l` (شمارنده خطوط) متصل می‌کند.

## 🎯 چالش هرج‌و‌مرج (The Chaos Challenge)

یک اسکریپت مانیتورینگ قدیمی در سرور دچار مشکل شده و لاگ‌های خطا با لاگ‌های معمولی مخلوط می‌شوند. شما باید این وضعیت را ساماندهی کنید.

**وظایف شما:**
۱. دستوری بنویسید که مسیر جاری شما را گرفته و در فایلی به نام `report.txt` ذخیره کند (محتوای قبلی باید پاک شود).
۲. بدون پاک شدن محتوای قبلی، متن `RHCSA Training` را به انتهای همان فایل اضافه کنید.
۳. دستوری بنویسید که تلاش کند محتویات پوشه‌ای ناموجود به نام `/not-exist` را لیست کند، اما خروجیِ خطای آن مستقیماً در فایلی به نام `missing.log` ذخیره گردد.
۴. با استفاده از `Pipe`، دستوری بنویسید که محتویات پوشه `/etc` را لیست کرده و با کمک دستور `grep` فقط خطوطی را خروجی بدهد که کلمه `network` در آن‌ها وجود دارد.
