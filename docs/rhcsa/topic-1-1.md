# 🐧 درس ۱.۱: دسترسی به Shell و اجرای دستورات

:::info یادداشت معمار ارشد
در دنیای Production، یک اشتباه کوچک در تایپ یک Command می‌تواند به قیمت از کار افتادن کل سیستم تمام شود. در کلاس من هیچ جایگاهی برای حدس زدن وجود ندارد؛ شما باید دقیقا بدانید هر کاراکتری که تایپ می‌کنید چه معنایی دارد.
:::

## 🧠 مفهوم اصلی (The Core Concept)

در محیط‌های Production سرورهای لینوکس معمولاً فاقد رابط گرافیکی (GUI) هستند تا منابع سیستم هدر نرود و امنیت بالاتر برود. تنها راه ارتباطی شما با سرور، **Shell** است. شل یک مفسر (Interpreter) است که دستورات شما را می‌گیرد و برای اجرا به هسته (Kernel) تحویل می‌دهد. فضایی که شما در آن منتظر تایپ دستورات هستید، Shell prompt نامیده می‌شود.

### ساختار دستوری (Syntax) در لینوکس
هر دستوری که در لینوکس وارد می‌کنید، از یک ساختار ثابت سه‌بخشی پیروی می‌کند:
```bash
Command [Options] [Arguments]

```

## 💻 لابراتوار عملی (Hands-on Lab)

ماشین مجازی لینوکس خود را روشن کنید و دستورات زیر را دقیقاً وارد کنید:

### ۱. چاپ متن در خروجی

```bash
echo "Welcome to RHCSA"

```

* **`echo`**: این Command است. کار آن چاپ کردن متن روی صفحه نمایش (Standard Output) است.
* **`"Welcome to RHCSA"`**: این Argument است که به دستور echo پاس داده‌اید.

### ۲. نمایش تاریخ و زمان سیستم (UTC)

```bash
date -u

```

* **`date`**: این Command تاریخ و زمان فعلی سیستم را نشان می‌دهد.
* **`-u`**: این یک Option است که زمان را بر اساس ساعت جهانی (UTC) نشان می‌دهد.

### ۳. بررسی نسخه Kernel

```bash
uname -r

```

* **`uname`**: این Command اطلاعات پایه‌ای سیستم‌عامل شما را چاپ می‌کند.
* **`-r`**: این Option مخفف release است و نسخه دقیق Kernel لینوکس را نشان می‌دهد.

### ۴. پاک‌سازی محیط کار

```bash
clear

```

* این Command صفحه ترمینال شما را پاک می‌کند تا محیط کار خلوت شود.

## ⚠️ چالش آشوب (The Chaos Challenge)

ماموریت شما:
۱. متنی با عنوان `Initiating Server Diagnostics` را روی صفحه چاپ کنید.
۲. نسخه دقیق Kernel سرور را استخراج کنید.
۳. تاریخ و زمان سرور را منحصراً به صورت زمان جهانی (UTC) بررسی کنید.
۴. تمام محتوای روی صفحه ترمینال را کاملاً پاک کنید.
