# ۱. معرفی دوره IaC و CI/CD (فاز صفر)

به عنوان Principal DevOps Architect، Lead SRE و ممتحن رسمی HashiCorp و فرآیندهای CI/CD، ساختار درخواستی شما را به طور کامل دریافت و تایید کردم.

هدف من در اینجا صرفاً آموزش نیست؛ بلکه انتقال یک تفکر مهندسی استاندارد (Mindset) و Enterprise به شماست. در این مسیر، به هیچ وجه روی پاسخ‌های آماده حساب نکنید. من شما را با سناریوهای واقعی و شکسته مواجه خواهم کرد تا مهارت Troubleshooting شما در سطح یک SRE واقعی تقویت شود.

در ادامه، معرفی جامع دوره و نقشه راه ما ارائه شده است.

## نام و سطح دوره

- **نام دوره:** Zero to Hero: Enterprise IaC & CI/CD Masterclass
- **سطح دوره:** از صفر مطلق تا سطح پیشرفته و معماری (Architect)، با تمرکز بر Best Practiceهای سازمانی.

## فلسفه وجودی و مفاهیم بنیادین

مفاهیم Terraform و CI/CD قلب تپنده چرخه حیات توسعه نرم‌افزار (SDLC) در دنیای مدرن هستند.

- **پلتفرم Terraform:** یک ابزار Infrastructure as Code (IaC) است که به شما اجازه می‌دهد زیرساخت ابری (Cloud Infrastructure) خود را به صورت کدهای Declarative (نظیر HCL) تعریف کنید.
- **مفهوم CI/CD:** اتوماسیون یکپارچه‌سازی مداوم (Continuous Integration) و استقرار مداوم (Continuous Deployment) است تا کدهای نوشته شده با امنیت و سرعت بالا وارد محیط Production شوند.

**فلسفه و چرایی:** این ابزارها برای حذف کامل خطای انسانی (Human Error)، ایجاد تکرارپذیری (Idempotency)، مدیریت یکپارچه پیکربندی و افزایش چشمگیر سرعت تحویل محصول (Time-to-Market) طراحی شده‌اند.

> **مثال دنیای واقعی:** به جای اینکه وارد کنسول AWS شوید و ساعت‌ها برای ساخت VPC، Security Groups و EC2 Instances کلیک کنید (رویکرد ClickOps)، صرفاً فایل `main.tf` را می‌نویسید و با اجرای یک دستور ساده در چند ثانیه هزاران سرور را در چندین Region می‌سازید:
>
> ```bash
> terraform apply -auto-approve
> ```
>
> سپس، بلافاصله پس از یک `git push`، پایپ‌لاین CI/CD شما (مثلاً GitHub Actions) به صورت خودکار کدها را Test کرده، Imageهای Docker را Build می‌کند و نسخه جدید را بدون هیچ قطعی (Zero-Downtime) روی زیرساخت جدید Deploy می‌کند.

## تاریخچه و تکامل زیرساخت‌ها

در گذشته، مدیران سیستم (SysAdmins) سرورها را به صورت فیزیکی درون رک‌ها نصب می‌کردند. با ظهور Cloud، رویه به سمت ساخت سرورها از طریق رابط کاربری گرافیکی (UI) شیفت پیدا کرد یا در بهترین حالت از Bash Scriptهای شکننده (Fragile) استفاده می‌شد. این رویکردها مستندسازی نداشتند و منجر به ایجاد سرورهایی می‌شدند که پیکربندی آن‌ها برای هیچ‌کس جز سازنده‌اش مشخص نبود (معروف به Snowflake Servers).

[Image of CI/CD pipeline workflow diagram]

امروزه همه چیز به Code تبدیل شده است (Everything as Code). ما زیرساخت را مانند یک نرم‌افزار توسعه می‌دهیم (GitOps)، برای آن Version Control داریم و هر تغییری قبل از اعمال، توسط Pipelineها بررسی و تایید می‌شود.

## شرح دوره

این یک Bootcamp کاملاً عملیاتی است. ما معماری زیرساخت را درک می‌کنیم، کدهای تمیز و ماژولار می‌نویسیم، Stateهای پیچیده Terraform را مدیریت می‌کنیم و Pipelineهای قدرتمند CI/CD را به همراه مفاهیم امنیتی (DevSecOps) و مدیریت هزینه (FinOps) روی GitHub Actions و GitLab CI پیاده‌سازی می‌کنیم.

هدف نهایی، تربیت یک مهندس ارشد است که توانایی طراحی، پیاده‌سازی و عیب‌یابی کل مسیر از مرحله Push تا Deploy را در مقیاس Enterprise داشته باشد.

## متدولوژی و سیستم کاری

روش آموزش من به شدت سخت‌گیرانه، تعاملی و بر پایه چالش است. روال هر مبحث دقیقاً به شکل زیر خواهد بود:

1. **مفهوم اصلی (The Core Concept):** توضیح شفاف و سریع مفهوم به همراه مثال کاربردی.
2. **آزمایشگاه عملی (Hands-on Lab):** ارائه کدهای HCL یا YAML استاندارد با تحلیل خط به خط (Line-by-Line Breakdown).
3. **چالش آشوب (The Chaos Challenge):** شما را در یک سناریوی بحرانی و خراب (نظیر State Conflict یا Pipeline Crash) قرار می‌دهم. شما باید با استفاده از CLI، مشکل را کشف و رفع کنید. من اشتباهات شما را با دقت و بدون ارفاق اصلاح خواهم کرد.
4. **مینی پروژه (Mini-Project):** در پایان ماژول‌های اصلی، یک پروژه اتوماسیون کامل برای اجرا خواهید داشت.

## پیش‌نیازها و ابزارهای مورد نیاز

**⚠️ هشدار جدی:** اگر مهارت‌های بنیادین زیر را ندارید، همین الان دوره را متوقف کرده و آن‌ها را یاد بگیرید. بدون این موارد، یادگیری IaC و CI/CD غیرممکن است.

### مهارت‌های الزامی

- **مبانی لینوکس (Linux Basics):** تسلط بر Shell، درک پرمیشن‌ها، مدیریت فایل و پردازش‌ها.
- **شبکه (Networking):** درک عمیق از TCP/IP، DNS، Routing و Load Balancing.
- **مفاهیم وب (Web Concepts):** شناخت پروتکل HTTP/HTTPS، مفهوم RESTful API و Webhook.
- **مبانی گیت (Git Basics):** تسلط بر دستورات اصلی (`commit`, `branch`, `merge`, `push`, `pull`).

### ابزارهای الزامی

- **محیط توسعه (IDE):** یک ویرایشگر استاندارد مانند VS Code.
- **ابزارهای خط فرمان:** نصب Git CLI و Terraform CLI روی سیستم.
- **اکانت‌ها:** داشتن حساب کاربری در GitHub یا GitLab، و دسترسی به یک Cloud Provider (نظیر AWS، GCP یا Azure - ترجیحاً اکانت‌های Free Tier برای تمرین).
