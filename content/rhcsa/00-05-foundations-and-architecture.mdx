# مفاهیم پایه، تاریخچه و معماری لینوکس

پیش از تعامل مستقیم با سیستم و اجرای دستورات، درک دقیق معماری لینوکس الزامی است. بدون شناخت ساختار زیرین، مدیریت سیستم به جای مهندسی، صرفاً به اجرای مکانیکی دستورات محدود خواهد شد.

## مفهوم اصلی و چرایی (The Core Concept)

هسته لینوکس در سال ۱۹۹۱ توسط Linus Torvalds به عنوان یک پروژه شخصی پایه‌گذاری شد و امروزه به سیستم‌عامل پیش‌فرض زیرساخت‌های حیاتی جهان تبدیل شده است. نسخه Red Hat Enterprise Linux (RHEL) توزیعی تجاری، بسیار پایدار و ایمن است که سازمان‌های بزرگ برای مدیریت زیرساخت‌های کلان خود از آن بهره می‌برند.

برای درک معماری لینوکس، می‌توان آن را به اجزای زیر تقسیم کرد:

- **هسته (Kernel):** هسته مرکزی سیستم‌عامل است که مستقیماً با سخت‌افزار (RAM, CPU, Disk) در ارتباط است. تعامل مستقیم کاربر با Kernel امکان‌پذیر نیست.
- **پوسته (Shell):** رابطی است که دستورات کاربر را دریافت کرده و به زبانی که Kernel قادر به درک آن باشد، ترجمه می‌کند. در این دوره، تمرکز بر روی پوسته پیش‌فرض یعنی `bash` خواهد بود.
- **رابط خط فرمان در برابر رابط گرافیکی (CLI vs GUI):** در محیط‌های لینوکس سازمانی، استفاده از رابط گرافیکی (GUI) به دلیل مصرف منابع غیرضروری مرسوم نیست. مدیریت سرورها منحصراً از طریق رابط خط فرمان (CLI) انجام می‌شود.
- **سلسله‌مراتب ریشه (Root Hierarchy - /):** بر خلاف سیستم‌عامل ویندوز که از درایوهای مجزا استفاده می‌کند، در لینوکس همه‌چیز (شامل فایل‌ها، دیسک‌ها، سخت‌افزارها و پردازش‌ها) از یک نقطه واحد به نام Root (ریشه) آغاز می‌شود که با علامت `/` نشان داده می‌شود. تمامی دایرکتوری‌ها از این نقطه منشعب می‌شوند.

## دیدگاه SRE و DevOps

در محیط‌های مقیاس‌پذیر و در چارچوب فرهنگ DevOps، مدیریت دستی و کلیک کردن روی رابط‌های گرافیکی برای هزاران سرور غیرممکن است. زیرساخت‌ها به صورت کد (Infrastructure as Code - IaC) مدیریت می‌شوند. ابزارهای اتوماسیون مانند Ansible یا پلتفرم‌هایی نظیر Kubernetes، کاملاً متکی به فایل‌های متنی و اجرای دستورات در سطح Shell هستند. درک عمیق از نحوه عملکرد Kernel و ساختار دایرکتوری‌ها (`/`)، امکان مانیتورینگ دقیق و اتوماسیون بی‌نقص را فراهم می‌سازد.

## آزمایشگاه عملی و اعتبارسنجی (Hands-on Lab & Verification)

در این بخش، بدون ایجاد تغییرات در پیکربندی، تنها به بازرسی سیستم پرداخته می‌شود.

برای بررسی نسخه Kernel، دستور زیر مورد استفاده قرار می‌گیرد:

```bash
uname -r
```

> **توجه:** آرگومان `-r` مخفف release است و نسخه دقیق Kernel در حال اجرا را نمایش می‌دهد.

برای بررسی Shell فعال در سیستم:

```bash
echo $SHELL
```

> **توجه:** خروجی استاندارد این دستور باید مسیر `/bin/bash` باشد.

برای مشاهده پایین‌ترین سطح سیستم (Root Hierarchy) و ساختار دایرکتوری‌های پایه:

```bash
ls -l /
```

> **توجه:** پرچم `-l` اطلاعات را به صورت long format و با جزئیات کامل نمایش می‌دهد. دایرکتوری‌هایی نظیر `/etc` و `/var` در پیکربندی‌های آتی بسیار پرکاربرد خواهند بود.

### اعتبارسنجی (Verification)

در سیستم‌عامل لینوکس، اجرای موفقیت‌آمیز بسیاری از دستورات با عدم نمایش خروجی همراه است (Silent Success). برای اطمینان از اجرای بدون خطای دستورات، از متغیر محیطی Exit Code استفاده می‌شود:

```bash
echo $?
```

خروجی `0` نشان‌دهنده اجرای موفقیت‌آمیز دستور قبلی است. هر عدد دیگری نمایانگر بروز خطا در اجرای دستور می‌باشد.

## نکات امتحانی و تله‌ها (Exam Gotchas & Pro Tips)

- **تله مرگبار ساختار دایرکتوری:** دایرکتوری ریشه سیستم (یعنی `/`) نباید با دایرکتوری خانگی کاربر root (یعنی `/root`) اشتباه گرفته شود. این اشتباه در آزمون‌های عملی منجر به قرارگیری فایل‌های حیاتی در مسیرهای نادرست شده و موجب از دست رفتن نمره خواهد شد.
- **محیط آزمون:** سرورهای ارائه‌شده در آزمون رسمی RHCSA فاقد محیط گرافیکی (GUI) هستند. مهارت کار با محیط متنی و خط فرمان برای موفقیت در آزمون الزامی است.

## چالش هرج و مرج (The Chaos Challenge)

**سناریو:** ورود به یک سرور ناشناس بدون هیچ‌گونه اطلاعات پیش‌زمینه.
**ماموریت:** با استفاده از فایل‌های موجود در دایرکتوری `/etc`، دستوری اجرا شود که نسخه دقیق سیستم‌عامل را نمایش دهد.

**راهنمایی:** در لینوکس، فایل‌های پیکربندی و اطلاعات هویتی سیستم در دایرکتوری `/etc` نگهداری می‌شوند. برای یافتن پاسخ، باید به دنبال فایلی بود که کلمه `release` در نام آن وجود داشته باشد و محتوای آن بررسی شود. این تمرین برای درک نحوه کاوش در فایل‌های سیستمی طراحی شده است.
