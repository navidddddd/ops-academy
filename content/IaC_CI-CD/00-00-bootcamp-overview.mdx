# معرفی بوت‌کمپ و مسیر یادگیری

ورود به دنیای مهندسی زیرساخت و قابلیت اطمینان سایت، نیازمند درک عمیق از معماری سیستم‌ها و ابزارهای اتوماسیون است. این بوت‌کمپ با رویکردی مبتنی بر یادگیری مفهومی طراحی شده است تا پیش از ورود به فاز عملیاتی و کدنویسی، پایه‌های تئوریک و مهندسی مستحکم شوند.

## فلسفه آموزشی (Zero to Hero: DevOps, IaC & SRE)

تبدیل شدن به یک مهندس خبره در سطح کلان، نیازمند درک دقیق «چرایی» استفاده از تکنولوژی‌هاست، نه صرفاً حفظ کردن و اجرای «چگونگی» دستورات. هدف اصلی این مسیر آموزشی، پرورش تفکر یک معمار سیستم برای طراحی، پیاده‌سازی و نگهداری زیرساخت‌هایی مقیاس‌پذیر، امن و کاملاً خودکار است.

## چرا IaC، CI/CD و GitOps؟

در محیط‌های مدرن توسعه نرم‌افزار، سرعت تحویل و پایداری سرویس‌ها دو رکن اساسی محسوب می‌شوند. برای دستیابی به این اهداف، تسلط بر سه مفهوم زیر حیاتی است:

- **ابزارهای زیرساخت به عنوان کد (Infrastructure as Code - IaC):** این ابزارها امکان مدیریت زیرساخت‌های فیزیکی و ابری را دقیقاً مشابه کدهای نرم‌افزاری فراهم می‌کنند. این رویکرد به معنای پایان یافتن پیکربندی‌های دستی، تکراری و مستعد خطاست.
- **خطوط لوله یکپارچه‌سازی و استقرار مداوم (CI/CD Pipelines):** فرآیندهای تست، بیلد و استقرار را به صورت کاملاً خودکار اجرا کرده و کیفیت کد را پیش از ورود به محیط Production تضمین می‌کنند.
- **رویکرد GitOps:** این پارادایم، مخازن Git را به عنوان «تنها منبع حقیقت» (Single Source of Truth) معرفی می‌کند. در این معماری، هرگونه تغییر در زیرساخت یا اپلیکیشن، منحصراً از طریق کامیت‌ها و Pull Requestها در Git اعمال و مدیریت می‌شود.

## تاریخچه و تکامل: از ClickOps تا اتوماسیون هوشمند

در گذشته‌ای نه چندان دور، مهندسان زیرساخت برای راه‌اندازی سرورها و شبکه‌ها نیازمند صرف ساعت‌ها زمان در پنل‌های کاربری تحت وب بودند؛ فرآیندی که اصطلاحاً ClickOps نامیده می‌شود. این روش به شدت کند، غیرقابل ردیابی و مستعد خطای انسانی بود.

با ظهور فرهنگ DevOps و ابزارهای اعلانی (Declarative) مانند Terraform، سطح جدیدی از اتوماسیون هوشمند شکل گرفت. در این مکانیزم، مهندس زیرساخت تنها «وضعیت مطلوب» (Desired State) را توصیف می‌کند و ابزارها به صورت خودکار مسیر و اقدامات لازم برای رسیدن به آن وضعیت را محاسبه و اجرا می‌کنند.

## اهداف آموزشی و خروجی دوره

در پایان این دوره آموزشی، مهارت‌های کلیدی زیر در سطح استاندارد سازمانی قابل ارائه خواهند بود:

- کدنویسی زیرساخت‌های پیچیده با استفاده از Terraform و مدیریت امن فایل‌های State در محیط‌های کار تیمی.
- طراحی و بهینه‌سازی خطوط لوله پیشرفته CI/CD با بهره‌گیری از ابزارهایی نظیر GitHub Actions و GitLab CI.
- پیاده‌سازی استراتژی‌های پیشرفته استقرار نرم‌افزار، از جمله استقرارهای Blue/Green و Canary.
- اجرای مفاهیم پیشرفته GitOps با استفاده از ابزارهایی نظیر ArgoCD بر روی کلاسترهای Kubernetes.
- پیاده‌سازی عملی اصول SRE شامل محاسبه شاخص‌های SLI/SLO و مدیریت بودجه خطا (Error Budgets).

## پیش‌نیازها و ابزارهای مورد نیاز

برای پیشبرد موفقیت‌آمیز فازهای عملیاتی این بوت‌کمپ، تامین الزامات زیر پیش از شروع دوره ضروری است:

- آشنایی با سیستم‌عامل Linux و کار در محیط خط فرمان (CLI).
- درک مفاهیم پایه سیستم کنترل نسخه Git (شامل فرآیندهای commit, push, pull و مدیریت branch).
- نصب و پیکربندی ابزارهای پایه شامل: ویرایشگر VS Code، موتور Docker، خط فرمان Git و Terraform CLI.

> **نکته حرفه‌ای (Pro Tip):** در محیط‌های سازمانی و پروژه‌های واقعی، همواره نسخه‌های ایمیج‌ها و ابزارهای مورد استفاده را قفل (Pin) کنید. استفاده از تگ `latest` می‌تواند منجر به بروز تغییرات غیرمنتظره، شکستن خطوط لوله و ایجاد اختلالات بحرانی در سرویس‌ها شود.

## تطابق با مدارک بین‌المللی

محتوای تدوین‌شده و سناریوهای عملی این دوره، بر اساس سرفصل‌های رسمی مدارک معتبر جهانی طراحی شده‌اند تا آمادگی لازم برای شرکت در آزمون‌های زیر فراهم گردد:

- گواهینامه HashiCorp Certified: Terraform Associate
- گواهینامه رسمی GitHub Actions
- آزمون‌های مرتبط با بنیاد CNCF و مدیریت پلتفرم Kubernetes

## مخاطبین هدف و زمان تخمینی

این بوت‌کمپ برای توسعه‌دهندگان نرم‌افزار، مدیران سیستم (SysAdmins) و متخصصین حوزه DevOps و SRE طراحی شده است. زمان تخمینی برای مطالعه دقیق مفاهیم، اجرای سناریوهای عملی و حل چالش‌های مهندسی (Chaos Challenges)، بین ۴۰ الی ۵۰ ساعت برآورد می‌شود.

> **هشدار محیط عملیاتی:** رسیدن به سطح تخصص در این حوزه نیازمند استمرار است. عبور از کارگاه‌های عملی بدون اجرای دقیق دستورات توصیه نمی‌شود. مهارت واقعی یک مهندس در زمان مواجهه با لاگ‌های خطا و عیب‌یابی در شرایط بحرانی (Troubleshooting) سنجیده می‌شود.
